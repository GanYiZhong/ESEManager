# 管理員資料庫更新指南

## 📚 資料庫說明

專案使用兩個主要資料庫：

| 資料庫 | 來源 | 用途 |
|--------|------|------|
| `ese_songs.db` | Git Repository | 所有歌曲檔案的完整列表、下載連結 |
| `ese_local.db` | 本地 TJA 檔案 | 日文標題、副標題等本地化資訊 |

## 🔄 資料流程

```
Git Repository → ese_scraper_git_v2.py → ese_songs.db
   ↓                                           ↓
下載 TJA 檔案                            上傳到伺服器
   ↓                                           ↓
本地 TJA 目錄 → build_local_db.py → ese_local.db
```

編輯 `version.txt`：

- 次要更新（新增部分歌曲）: `1.0.0 → 1.1.0`
- 修正更新（bug 修復）: `1.0.0 → 1.0.1`

## 📊 資料庫結構

### ese_songs.db（V2 結構）

**三個主要表：**

1. **categories** - 分類表
   - `id`: 分類 ID
   - `name`: 分類名稱（如 "01 Pop"）
   - `path`: 分類路徑

2. **songs** - 歌曲主表
   - `id`: 歌曲 ID
   - `category_id`: 所屬分類
   - `song_name`: 歌曲名稱（不含副檔名）
   - `base_path`: 基礎路徑

3. **song_files** - 歌曲檔案表
   - `id`: 檔案 ID
   - `song_id`: 所屬歌曲
   - `filename`: 檔案名稱
   - `file_path`: 完整路徑
   - `file_type`: 檔案類型（.tja, .ogg 等）
   - `size`: 檔案大小
   - `download_url`: 下載連結

### ese_local.db

**主要表：**

**local_songs** - 本地歌曲資訊
- `id`: 記錄 ID
- `title`: 英文標題
- `title_ja`: 日文標題 ⭐
- `subtitle`: 英文副標題
- `subtitle_ja`: 日文副標題 ⭐
- `category`: 分類
- `bpm`: BPM
- `wave_file`: 音樂檔案名稱
- `file_path`: TJA 檔案路徑

## 🛠️ 進階管理

### 定期更新建議

建議更新頻率：
- **ese_songs.db**: 每週或當 Git 有大量新增時
- **ese_local.db**: 當下載新的 TJA 檔案後
- **version.txt**: 每次更新資料庫時

### 自動化腳本（進階）

可以設定 Windows 排程執行：

```bash
# 創建排程任務（每週日凌晨 2 點執行）
schtasks /create /tn "更新ESE資料庫" /tr "C:\Users\User\Desktop\ESEManager\update_databases.bat" /sc weekly /d SUN /st 02:00
```

### 備份資料庫

```bash
# 創建備份目錄
mkdir backups

# 備份當前資料庫
copy ese_songs.db backups\ese_songs_%date:~0,10%.db
copy ese_local.db backups\ese_local_%date:~0,10%.db
```

## ⚠️ 常見問題

### Q1: Git Clone 失敗怎麼辦？

**解決方案：**
```bash
# 方法 1: 使用 fast 版本（不需要 Git）
python ese_scraper_fast_v2.py

# 方法 2: 手動刪除舊的 clone 目錄
rmdir /s /q ESE_clone
python ese_scraper_git_v2.py
```

### Q2: 本地 TJA 目錄不存在

**解決方案：**
```bash
# 先下載 TJA 檔案到本地
python download_songs.py --all --types .tja

# 然後指定正確的下載目錄
python build_local_db.py --dir "Downloads"
```

### Q3: 資料庫檔案太大無法上傳

**解決方案：**
```bash
# 檢查資料庫大小
dir ese_songs.db
dir ese_local.db

# 清理舊資料（如需要）
python query_songs_v2.py --stats

# 如果確實太大，考慮分割上傳或壓縮
```

### Q4: 如何確認資料庫已成功更新？

```bash
# 檢查檔案修改時間
dir ese_songs.db
dir ese_local.db

# 檢查資料庫統計
cd archive\source_code
python query_songs_v2.py --stats
```
