# ESE Scraper 修復說明

## 📅 修復日期
2025-12-17

## 🐛 問題描述

### 1. 遞迴掃描問題
`ese_scraper_fast_v2.py` 在執行時會卡住，只掃描分類的根目錄，沒有進入子目錄：

```
✓ 01 Pop: 657 資料夾, 0 新檔案, 1 跳過
✓ 02 Anime: 540 資料夾, 0 新檔案, 1 跳過
...
```

**原因：**
- `process_directory()` 方法只處理當前目錄的檔案
- `scan_subdirectories()` 方法只掃描一層子目錄
- 沒有真正的遞迴處理所有層級的資料夾

### 2. Git URL 過時
原始 URL `https://git.vanillaaaa.org/ESE/ESE.git` 需要更新為新地址。

---

## ✅ 修復內容

### 修復 1: 重寫遞迴邏輯

**檔案：** `ese_scraper_fast_v2.py`

**修改位置：** `process_directory()` 方法（第 240-324 行）

**修改內容：**

1. 新增 `recursive` 參數，預設為 `True`
2. 在處理檔案後，自動遞迴處理所有子目錄：

```python
# 遞迴處理子目錄
if recursive and folders:
    for folder in folders:
        folder_path = folder.get("path", "")
        if folder_path:
            sub_stats = self.process_directory(folder_path, category_id, "", recursive=True)
            local_stats["folders"] += sub_stats["folders"]
            local_stats["files"] += sub_stats["files"]
            local_stats["skipped"] += sub_stats["skipped"]
```

3. 簡化 `scrape()` 方法，移除多餘的 `scan_subdirectories()` 呼叫
4. 修改統計更新邏輯，只在頂層更新全局統計

**效果：**
- ✅ 現在會遞迴掃描所有層級的子目錄
- ✅ 不會再卡住
- ✅ 能正確統計所有檔案

## ⚠️ 注意事項

1. **首次執行時間**
   - Git 方式：1-3 分鐘
   - API 方式：5-15 分鐘（取決於網路速度和線程數）

2. **網路要求**
   - 需要穩定的網路連線
   - 建議使用有線網路或穩定的 Wi-Fi

3. **資料庫大小**
   - `ese_songs.db`：約 2-3 MB
   - `ese_local.db`：約 1-2 MB

4. **增量更新**
   - 再次執行會自動跳過已存在的檔案
   - 只會下載新增的內容
